Java反射-Class类

发表于 2017-10-02 | 分类于 JavaSE | 阅读次数

接着上一篇Java反射-是什么？而来，其实都围绕Class类这个主题而来，这两篇基本完全照搬廖雪峰大神的Java教程而来。(￣▽￣)~*不能掌握一手资料，英语渣渣的我，愿有朝一日可阅读源码，可看懂英文的专业书籍，奈何时间不等人，还未了解为什么，就照搬接受——是什么就是什么，吸收多少未知，会用多少未知……

★Class类，我忍你很久了，你就不能换个样子？

①如何获取一个class的Class实例？

有三种方式：

  • Type.class

  • getClass()

  • Class.forName()

②Class实例在JVM中是唯一的？

  • 可以用==比较两个Class实例

③Class实例比较和instanceof的差别？

instanceof 不但匹配当前类型，而且匹配父类也是可以的。而==，能精确判断该类型，不能做子类的比较

④反射的目的？

反射的目的是当获得某个Object实例时，我们可以获取该Object的class信息，如调试当前某个实例变量时，eclipse能通过反射获取其具体的class信息，如其类型，当前的值……

⑤从Class实例获取class信息？

  • getName()
  • getSimpleName()
  • getPackage()

⑥从Class实例判断class类型？

  • isInterface()
  • isEnum()
  • isArray()
  • isPrimitive()

可见都是Class实例.is……。可是int.class是什么鬼？怎么基本类型也是class？逗比吧？虽然基本类型变量本身不是class，但是JVM的内部会创建int.class这种基本类型对应的Class实例。

⑦创建class实例？（注意是小头的class不是大头的Class)

  • newInstance()

可这样做有局限，只能调用String类无参数的默认构造方法，带参数的构造方法是没办法通过这个方法来调用的

⑧动态加载？

利用JVM动态加载class的特性：

  • 可以在运行期根据条件不同加载不同的实现类
  • 利用JVM动态加载class的特性，可以在运行期根据条件加载不同的实现

这里的关键在于Class.forName()能否找到这个类，只要将Log4j的jar包通过ADD JARs…放到Referenced Libraries下，接着将log4j的配置文件放到classpath（所谓的src）的根目录中就找的着，找不到就捕获这个异常。==》存在这个类吗，存在就使用你吧，不存在那就使用默认的好了

⑨总结？

  • JVM为每个加载的class创建对应的Class实例来保存class 的所有信息
  • 获取一个class对应的Class实例后，就可以获取该 class 的所有信息
  • 通过Class实例获取class信息的方法称为反射(Reflection)
  • JVM总是动态加载class ，可以在运行期根据条件控制加载 class
