①话说，你经常定义一个类，可你有了解过class吗？

​ 我一直都有个疑问，interface到底是不是一个class，廖大说：“广义上来说是个class”，而且说到“Java除了基本类型外其它都是class(包括那100%抽象的interface)”。像是String、Object……

class的本质是数据类型，当我们new一个对象的时候，必须严格按照数据类型来赋值，而且无继承关系的数据类型无法赋值，如：(Student是Person的子类)

Person p = new Person()是正确的，Person s = new Student()也是可以的，而String str = new Person()就不行了，这会编译报错，毕竟Java是强类型语言。

②class/interface也有数据类型？

​ 它们的数据类型就是Class，是不是很surprised！不是惊讶，而是懵逼了，class还有类型啊，就是这个跟我长得一模一样，就是头大了一点的Class，嗦嘎，class的本质是数据类型，像是String……你凭啥说String这是个类啊，就因为它public final class String这里有个class关键字？对头。

⑥小结？

  • JVM为每个加载的class创建对应的Class实例，并在该实例中保存该class的所有信息
  • 如果获取了某个Class实例，则可以获取到该实例对应的class的所有信息
  • 通过Class实例获取class信息的方法称为反射（Reflection）
